# /impeccable critique

Two independent assessments, synthesized into a structured report. No fixes — only diagnosis and prioritized recommendations.

## Process

**Assessment A — Design Review (LLM)**
Read the relevant code and identify:
- Visual hierarchy issues
- Information architecture problems
- Spacing and rhythm inconsistencies
- Typography issues
- Color and contrast failures
- Component vocabulary inconsistencies

**Assessment B — Anti-Pattern Detection**
Scan for 41 known AI/slop tells:
- Gradient text, decorative glassmorphism, aurora backgrounds
- Identical card grids without semantic purpose
- Tiny uppercase tracked eyebrows on every section
- Side-stripe borders as decoration
- Bounce/elastic animation curves
- Inter/Poppins as unreflective default
- Purple-to-blue gradient color palettes
- Numbered section markers as scaffolding

## Guardrails

- Do not soften criticism ("consider exploring" → name the specific fix)
- Do not ask generic audience questions — only clarifications tied to actual findings
- Do not recommend commands outside the 25 available `/impeccable` commands
- Assessment A completes before Assessment B
- Detector findings anchor synthesis, not vice versa
